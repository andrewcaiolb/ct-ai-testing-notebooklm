# 🧠 CT-AI Testing – Aprendizado com NotebookLM

## 📌 Contexto

Este projeto foi desenvolvido como desafio no bootcamp CI&T - Do Prompt ao Agente, com o objetivo de utilizar IA como ferramenta de aprendizado ativo.

O foco do estudo com o NotebookLM é aprender as principais abordagens para a certificação **CT-AI (Certified Tester AI Testing)**, abordando conceitos fundamentais de testes em sistemas baseados em Inteligência Artificial.

---

## 🎯 Objetivos

* Compreender os fundamentos de AI Testing com base no syllabus CT-AI
* Estudar e consolidar os principais conceitos exigidos para a certificação CT-AI (BSTQB)
* Identificar riscos em sistemas de IA (bias, drift, data poisoning)
* Utilizar o NotebookLM como ferramenta de aprendizado ativo
* Praticar engenharia de prompts para aprofundamento técnico

## 📘 Miniguia de Estudo

### 🧾 Resumo

AI Testing envolve validar sistemas que aprendem com dados e apresentam comportamento não determinístico.

Principais desafios:

* Dependência de dados
* Resultados variáveis
* Falta de previsibilidade total

Principais riscos:

* Bias (viés nos dados)
* Drift (mudança ao longo do tempo)
* Data Poisoning (dados maliciosos)

---

### 📖 Glossário

* **Bias (Viés):** Tendência injusta nos dados
* **Drift:** Mudança nos dados ao longo do tempo
* **Data Poisoning:** Inserção de dados maliciosos
* **Modelo Probabilístico:** Resultado pode variar
* **Overfitting:** Modelo decorou os dados

---

* Liste riscos e estratégias de teste para IA como serviço
* Compare teste tradicional vs IA
* Explique como validar um modelo de IA
