O foco do estudo no notebookLM é a certificação **CT-AI (Certified Tester AI Testing)**, abordando conceitos fundamentais de testes em sistemas baseados em Inteligência Artificial.

## 4. Engenharia de Prompts e Cicatrizes

* Explique o que é AI Testing com base nas fontes

---

### 4.2 Evolução dos Prompts

Inicial:
O que é bias?

Refinado:

Avançado:
Explique bias em IA, riscos associados e impactos em sistemas reais e como isso afeta testes de software

---

### 4.3 Exemplo prático

**Pergunta:**
Explique o que é bias em IA

**Resumo da resposta:**
Bias em IA é uma diferença sistemática nos resultados de um modelo, podendo gerar decisões injustas. Pode ocorrer por problemas nos dados, no algoritmo ou por fatores humanos.

**Aprendizado:**
O viés pode comprometer diretamente a confiabilidade de sistemas de IA, principalmente quando os dados de treinamento não representam a realidade.

**Ajuste realizado:**
Foi necessário solicitar exemplos práticos para melhor compreensão do impacto em cenários reais.

## 5. Miniguia de Estudo

### 5.1 Resumo

AI Testing envolve validar sistemas que aprendem com dados e apresentam comportamento não determinístico.

Principais desafios:

* Dependência de dados
* Resultados variáveis
* Falta de previsibilidade total

Principais riscos:

* Bias (viés)
* Drift (mudança ao longo do tempo)
* Data Poisoning

---

### 5.2 Glossário

* **Bias:** Tendência injusta nos dados
* **Drift:** Mudança no comportamento do modelo
* **Data Poisoning:** Inserção de dados maliciosos
* **Modelo Probabilístico:** Resultado pode variar
* **Overfitting:** Modelo decorou os dados

---

### 5.3 Principais aprendizados

* Testes em IA são centrados em dados
* Sistemas exigem validação contínua
* A qualidade dos dados impacta diretamente o modelo
* Engenharia de prompts melhora o aprendizado
